◆ Step 3a — Row Matching Method
Choose one or more columns that together uniquely identify each row in both files. Use a composite key (2+ columns) when no single column is unique on its own.
Rows are matched by position — row 1 in Before is compared to row 1 in After, row 2 vs row 2, and so on. No unique key column is required. Rows beyond the shorter file's length will be reported as unmatched.
🔄 Sort Both Files Before Comparing (recommended)
Select a column to sort both files by before comparing. This ensures row 1 aligns with row 1 correctly. Sorting applies to the entire row.

📂 Step 3b — Map Comparison Columns
Map each Before column to its matching After column. These columns will be compared for each matched row. The key column above is used for matching only and does not need to be listed here.
📘 Before Column → 📚 After Column
📋 Step 4 — Comparison Conditions
Define how values are compared. Toggle on/off. Use * in Fields to apply a condition to all columns, or list specific column names separated by commas.
📊 Step 4 — Define Groups
Define one or more groups. Each group filters rows where all conditions match, then sums all numeric columns. You can add multiple filters per group (AND logic).
